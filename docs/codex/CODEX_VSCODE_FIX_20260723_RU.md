# Аудит и восстановление Codex в VS Code 2026-07-23

Статус: `CODEX_VSCODE_EXTENSION_RESTARTED_NO_DELETE`.

## Симптом

В панели VS Code Codex было сообщение: `Работа Codex неожиданно остановлена`. Кнопки панели предлагали открыть `Config.toml` или перезапустить Codex.

## Причина

Отдельный сервер расширения VS Code `openai.chatgpt-26.715.31925...\codex.exe app-server` не был запущен. Основное настольное приложение Codex при этом работало отдельно через пакет `OpenAI.Codex_26.715.10079.0` и процессы `ChatGPT.exe`/`codex.exe`.

Причина состояния связана с предыдущей мягкой разгрузкой: VS Code extension server был остановлен, чтобы снять нагрузку. Для работы панели VS Code его нужно не держать остановленным, а запускать и прижимать по приоритету.

## Что сделано

- Через UI Automation найдена и нажата штатная кнопка `Перезапустить` в панели VS Code Codex.
- Новый процесс расширения поднялся: `C:\Users\007\.vscode\extensions\openai.chatgpt-26.715.31925-win32-x64\bin\windows-x86_64\codex.exe`.
- Приоритеты `ChatGPT`/`codex` выставлены в `Idle`.
- Удалений не было.
- Папки `STAS*`, включая `STAS5` и `STAS6`, не изменялись.
- `C:\Users\007\.codex\config.toml` не изменялся.

## Контроль

После перезапуска VS Code-сервер Codex кратко прочитал диск и затем успокоился: контрольный срез показал `codex#1` с `0% CPU`, `0` операций чтения/записи в секунду и около `145 MB` памяти. Аварийная надпись в панели больше не находилась; вместо нее отображался рабочий экран Codex.

Высокие показатели `ChatGPT.exe` во время проверки относятся к активной текущей сессии Codex Desktop и запуску диагностических команд. Этот процесс нельзя безопасно убивать без разрыва текущего диалога.

## Правило дальше

Не останавливать VS Code OpenAI/Codex extension server, если нужна рабочая панель Codex в VS Code. Для разгрузки использовать низкий приоритет и исключения watcher/search/Pylance, а постоянное отключение расширения делать только отдельным решением пользователя.
