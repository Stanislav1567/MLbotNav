# Разгрузка CPU проекта 2026-07-23

Статус: `PASS_CODEX_PROJECT_CPU_RELIEF_APPLIED_NO_DELETE_NO_TRAINING`.

## Причина

Нагрузка шла не от STAS5 и не от обучения. Активных `python.exe` не было. Процесс Codex Desktop циклически запускал `git hash-object`, `git diff --no-index` и `git config` по неигнорируемым сгенерированным файлам `STAS4_FEATURE_HYPOTHESIS_REVIEW`. Повторное чтение тех же PNG/CSV/JSON дополнительно нагружало Microsoft Defender.

До исправления `codex.exe` использовал около `9%` всего CPU, Defender - до `8.5%`. В пятисекундном окне появилось более двадцати краткоживущих Git-процессов.

## Что изменено

- В `.gitignore` добавлены только сгенерированные подпапки и верхние PNG из `STAS4_FEATURE_HYPOTHESIS_REVIEW`.
- Верхние `README_RU.md` и итоговый Markdown не скрыты от Git.
- В `.vscode/settings.json` вся папка STAS4 review исключена из watcher/search/Pylance.
- Текущим процессам `ChatGPT`, `codex`, `Code`, `node_repl` и `codex-code-mode-host` выставлен приоритет `Idle`.
- Файлы и артефакты не удалялись. Обучение и forward не запускались.

## Контроль

- STAS4 в неигнорируемом Git-хвосте: было `258` файлов, осталось `2` верхних Markdown-файла.
- Общий неигнорируемый хвост: `170` файлов. Это преимущественно исходники, тесты и рабочие документы; они намеренно не скрывались.
- `codex.exe`: примерно `9% -> 0.03%` CPU в контрольном окне.
- Общий CPU после стабилизации: `10.0..13.6%`, среднее `12.3%`.
- Активных Python/ML-процессов нет.
- `.git/index.lock` отсутствует.

## Ограничение

Приоритет процессов Windows сбросится после перезапуска Codex/ПК. Git/VS Code исключения постоянные и останутся. Если после перезапуска снова появится нагрузка, достаточно повторно применить мягкое понижение приоритета из `docs/codex/commands.md`; отключать Defender или удалять артефакты не требуется.
