# Решение по FIBO в блоке `structure_ta`

Дата UTC: `2026-06-03T17:45:00Z`.

## Решение

FIBO возвращаем в блок `structure_ta` как нормальный технический блок, а не как две фиксированные фичи 0.382 и 0.618.

Сейчас в коде реально считаются только:

1. `fib_0382_distance`.
2. `fib_0618_distance`.

Этого мало для полноценной проверки гипотезы Фибо.

## Как должна работать Фибо

Фибо не рисуем руками. Ее должен натягивать алгоритм:

1. Берется рабочее окно `fib_window`.
2. Внутри окна ищется локальный минимум и максимум.
3. Для `long_only` сетка строится от low к high.
4. Для `short_only` сетка строится зеркально от high к low.
5. По каждому уровню считается дистанция текущей цены до уровня.
6. Дальше Optuna/калибровка проверяет, какой уровень, окно и допуск дают лучший результат.

## Уровни

Минимальная рабочая сетка retracement:

1. `0.236`.
2. `0.382`.
3. `0.500`.
4. `0.618`.
5. `0.786`.

Расширения для целей:

1. `1.272`.
2. `1.618`.
3. `2.618`.

## Что калибровать

Сам факт "натянуть Фибо" не калибруется: это расчетная процедура.

Калибруются параметры вокруг нее:

1. `fib_window` - окно, на котором ищем low/high.
2. `fib_level` - какой уровень лучше работает.
3. `threshold_fine` - допуск близости цены к уровню.
4. long/short отдельно, потому что сетка зеркальная.

## Что фиксируем

1. Старый параметр `fib_level` нужно вернуть в текущую матрицу.
2. В `structure_ta` нужно расширить Фибо с двух уровней до полноценной сетки.
3. В отчетах по блоку нужно показывать лучший уровень Фибо и лучший `fib_window`.
4. До реализации этого пункта блок `structure_ta` считается неполным по FIBO-логике.

## Практическое правило

Фибо участвует в блоковой калибровке `structure_ta` вместе с остальными параметрами блока.

`feature_sweep/Hxxx` не является заменой этой логики.
