## 0. Главный принцип

### 0.1. Разделение контуров

Optuna и ML работают как два разных контура.

1. Optuna калибрует параметры паспортов, action gates и OOS-результаты.
2. ML читает только утвержденные calibration packages.
3. Автоматической передачи из Optuna в ML пока нет.
4. Передача в ML только вручную через статус `APPROVED_FOR_ML`.

Готово, когда: это правило зафиксировано в статусе и соблюдается в следующих этапах.

### 0.2. Запрет старой каши

Запрещено:

1. Давать ML читать старые Optuna папки напрямую.
2. Давать ML читать `NO_GO`.
3. Давать ML читать `VALIDATION_FAIL`.
4. Давать ML читать CSV без паспортного контекста.
5. Давать ML читать `raw/quarantine` день как clean input.

Готово, когда: эти запреты есть в контракте и проверках.
