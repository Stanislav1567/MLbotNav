# ТЗ: аудит калибруемых ручек инструментов

Дата UTC: `2026-06-11T10:47:35Z`.

Статус:
`ACTIVE_CALIBRATION_PREP_TZ`.

## Цель

Разобрать каждый блок, фичу, индикатор и гипотезу как отдельный инструмент и зафиксировать:

1. Что у инструмента есть по умолчанию.
2. Что у инструмента можно крутить.
3. Что уже калибруется в текущих конфигах.
4. Какие signal-level линии/пороги отсутствуют.
5. Что нужно добавить или оставить как есть.
6. Как это должно работать отдельно для `LONG` и `SHORT`.

Это подготовительный аудит. Optuna/APTuna runtime не запускать.

## Главное Правило

Двигаемся строго блоками:

```text
Block 01 -> Block 02 -> Block 03 -> Block 04 -> Block 05 -> Block 06
```

Для каждого блока:

1. Пишем короткое ТЗ блока.
2. Выписываем все инструменты блока.
3. Для каждого инструмента разбираем расчетные параметры.
4. Для каждого инструмента разбираем signal-level параметры: линии, уровни, пересечения, силу сигнала, фильтры.
5. Сверяем, что уже есть в конфиге.
6. Сверяем, что реально применяется в коде.
7. Фиксируем, что добавить, что оставить, что не калибровать.
8. Только после фиксации блока переходим к следующему блоку.

## Что Сейчас Не Делаем

1. Не запускаем калибровку.
2. Не запускаем `C001 medium`.
3. Не запускаем forward/OOS.
4. Не принимаем production-решения.
5. Не правим конфиги до согласования ручек блока.
6. Не берем задачи из старой хронологии.

## Единица Аудита

Не просто `RSI` или `MACD`, а:

```text
инструмент + тип сигнала + направление
```

Примеры:

```text
RSI oversold rebound LONG
RSI overbought reject SHORT
MACD bull cross LONG
MACD bear cross SHORT
ATR volatility filter BOTH
```

Так мы не смешиваем расчет индикатора, линию сигнала, силу сигнала и направление сделки.

## Шаблон Разбора Инструмента

Для каждого инструмента фиксируем:

1. Блок.
2. Инструмент.
3. Тип: фича, индикатор, гипотеза, фильтр, риск/выход.
4. Русское название.
5. Что инструмент измеряет.
6. Дефолтные настройки.
7. Расчетные параметры: period/window/lookback/fast/slow/signal/smoothing.
8. Signal-level параметры: уровни, линии, пересечения, зоны, минимальная сила.
9. `LONG` логика.
10. `SHORT` логика.
11. Фильтры и подтверждения.
12. Что сейчас есть в конфиге.
13. Что реально используется в коде.
14. Чего не хватает.
15. Предлагаемые `min/max/step/values`.
16. Решение: оставить, добавить, проверить, не калибровать.

## Очередь Блоков

### Block 01: `price_volatility`

Русское название:
`Цена и волатильность`.

Инструменты:

1. `ret_1`, `ret_3`, `ret_6`, `ret_12`, `ret_24`.
2. `hl_spread`.
3. `rolling_std_20`.
4. `atr14`.

Главный вопрос блока:
сейчас калибруются окна расчета, но нужно разобрать signal-level пороги:

1. Порог силы изменения цены.
2. Минимальная/максимальная волатильность.
3. ATR min/max regime.
4. ATR multiplier для risk/SL/TP, если применимо.

### Block 02: `trend_momentum`

Русское название:
`Тренд и импульс`.

Инструменты:

1. EMA gap/slope/EMA200.
2. RSI.
3. MACD.
4. ADX.
5. Stochastic.

Главный вопрос блока:
сейчас калибруются periods, но нужно разобрать линии сигнала:

1. RSI lower/mid/upper.
2. MACD zero-line, histogram strength, cross threshold.
3. Stochastic lower/upper, K/D cross.
4. ADX threshold уже есть, но нужно проверить диапазоны.
5. EMA gap threshold уже есть как `min_abs_ema_gap`, но нужно проверить LONG/SHORT смысл.

### Block 03: `volume_flow`

Русское название:
`Объем и поток`.

Инструменты:

1. `vol_chg`.
2. `vol_z`.
3. `delta_volume`.
4. `obv_slope_5`.
5. `mfi14`.
6. `vwap_distance`.

Главный вопрос блока:
нужно разобрать отдельные пороги объема:

1. Volume spike threshold.
2. Volume dry-up threshold.
3. MFI zones.
4. VWAP distance threshold.
5. OBV/delta confirmation threshold.

### Block 04: `density_profile`

Русское название:
`Профиль плотности`.

Инструменты:

1. VPOC distance/share.
2. Bin share.
3. Cluster share.
4. VPOC drift.
5. Cluster ratio 60/240.

Главный вопрос блока:
нужно разобрать уровни плотности как signal-level зоны:

1. Дистанция до VPOC.
2. Минимальная доля кластера.
3. HVN/LVN реакция.
4. Acceptance/rejection зоны.

### Block 05: `structure_ta`

Русское название:
`Структура ТА`.

Инструменты:

1. Support/resistance distance.
2. Level strength.
3. Position in range.
4. Trend channel position.
5. FIBO.
6. Breakout/false breakout/retest.
7. Swing/BOS/CHOCH.
8. TP/SL/RR context.

Главный вопрос блока:
много линий и уровней должны быть явными ручками:

1. Дистанция до уровня.
2. Сила уровня.
3. Breakout threshold.
4. Retest window.
5. FIBO levels.
6. RR minimum.
7. Channel/range boundaries.

### Block 06: `pattern`

Русское название:
`Паттерны`.

Инструменты:

1. Свечные паттерны.
2. Дивергенции RSI/MACD/OBV.
3. Фигурные паттерны.
4. Pattern + level + volume entry.
5. Pattern SL/TP helpers.

Главный вопрос блока:
нужно разобрать параметры каждого паттерна:

1. Размер тела/тени.
2. Минимальная сила паттерна.
3. Age cap.
4. Количество касаний.
5. Breakout/retest/level/volume confirm.
6. SL buffer.
7. TP ladder.

## Выход По Каждому Блоку

После разбора блока должен быть короткий итог:

```text
BLOCK_XX_STATUS:
1. Что уже калибруется.
2. Что реально применяется.
3. Что не хватает.
4. Какие ручки согласованы добавить.
5. Какие ручки не трогаем.
6. Какие риски/вопросы остались.
7. Следующий блок.
```

## Финальный Выход Всего Аудита

После Block 06 должен быть общий реестр:

1. Все инструменты.
2. Все текущие параметры.
3. Все недостающие signal-level параметры.
4. Все hardcoded линии, которые стоит вынести в профили.
5. Решение, что добавлять в конфиги.
6. Только после этого можно возвращаться к `CASCADE_BLOCK_CALIBRATION`.
