## Дополнение 2026-06-04: фигурные паттерны в связке с объемом и локальными уровнями ТА

Фиксируем как рабочую гипотезу для доработки блока `pattern` и связки с блоками `structure_ta` и `volume_flow`.

Фигурный паттерн сам по себе не должен считаться достаточным сигналом на вход. Его нужно проверять в связке:

1. `pattern` - сама фигура рынка.
2. `structure_ta` - локальный уровень, структура, пробой, ретест, FIBO, support/resistance.
3. `volume_flow` - подтверждение объемом, delta/OBV/MFI/volume z-score.

Составной контур называем: `pattern_structure_volume_entry`.

Русское название: `Фигурный паттерн + уровень + объем для входа`.

Фигурные паттерны, которые нужно рассматривать:

1. Двойная вершина.
2. Двойное дно.
3. Голова и плечи.
4. Перевернутая голова и плечи.
5. Треугольник.
6. Вымпел.
7. Восходящий клин.
8. Нисходящий клин.
9. Диапазон.

Логика входа:

1. Сначала определяется фигура.
2. Затем проверяется локальный уровень из `structure_ta`.
3. Затем проверяется подтверждение объемом из `volume_flow`.
4. Только после связки `фигура + уровень + объем` формируется сигнал на вход.
5. `long_only` и `short_only` калибруются отдельно.

Параметры для будущей калибровки:

1. `pattern_window` - окно поиска фигуры.
2. `min_touches` - минимальное количество касаний/подтверждений границ фигуры.
3. `breakout_threshold` - насколько цена должна выйти за уровень.
4. `retest_window` - сколько баров ждать ретест.
5. `level_distance_threshold` - допустимая дистанция до локального уровня.
6. `volume_confirm_threshold` - порог подтверждения объемом.
7. `vol_z_window` - окно оценки относительного объема.
8. `sl_buffer` - запас для стопа.
9. `tp_ladder` - лесенка тейк-профитов.

Выход:

1. `SL` ставится за invalidation-точку: за границу фигуры, локальный уровень, FIBO или ATR-buffer.
2. `TP1` - ближайший локальный уровень.
3. `TP2` - measured move фигуры.
4. `TP3` - FIBO extension, следующий уровень или RR target.

Важно: этот контур не заменяет блоки `pattern`, `structure_ta`, `volume_flow`, а калибруется как связка инструментов для поиска входа.

До реализации фигурных паттернов блок `pattern` считается неполным по части классических графических фигур: сейчас он в основном покрывает свечные паттерны и дивергенции.

## Дополнение 2026-06-04T14:17:45Z: режим CASCADE_BLOCK_CALIBRATION

Фиксируем новый рабочий режим калибровочного узла: `CASCADE_BLOCK_CALIBRATION`.

Правило режима:

1. Один блок = один каскад.
2. `LONG` и `SHORT` калибруются отдельно и не смешиваются.
3. `wide` всегда запускается первым и смотрит широкий диапазон блока.
4. `medium` запускается только вокруг лучшего `tradeful` кандидата из `wide`.
5. `narrow` запускается только вокруг лучшего `tradeful` кандидата из `medium`.
6. Если на `wide` нет `tradeful` кандидата, `medium/narrow` не сужаются вслепую: фиксируем `NO_TRADEFUL_CANDIDATE` и переходим к следующему блоку.
7. Если `tradeful` кандидат есть, но он минусовой, каскад все равно продолжается, чтобы понять лучший возможный результат блока.
8. Если появляется плюсовой кандидат, он сохраняется в `positive/top candidates` как тестовый кандидат, без production-перевода.

Назначение режима:

1. Не калибровать все 68-69 фичей сразу.
2. Не запускать полный взрывной перебор всех комбинаций.
3. Идти блоками: блок отдельно, сторона отдельно, каскад отдельно.
4. Проверять, что параметры блока реально доходят от min до max и затем сужаются вокруг найденного результата.
5. Сохранять читаемый русский итог по каждому блоку: что калибровали, какие сетки прошли, какой лучший результат, есть ли кандидат.
