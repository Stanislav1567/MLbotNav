# Приложение к ТЗ: фигурный паттерн + уровень + объем

Дата UTC: `2026-06-04T00:00:00Z`.

## Идея

Фигурный паттерн не используем как одиночный сигнал.

Сигнал на вход должен формироваться связкой:

`фигура рынка + локальный уровень ТА + подтверждение объемом`.

Техническое имя контура: `pattern_structure_volume_entry`.

Русское имя: `Фигурный паттерн + уровень + объем для входа`.

## Какие блоки участвуют

1. `pattern` - определяет фигуру.
2. `structure_ta` - дает уровень, пробой, ретест, FIBO, support/resistance.
3. `volume_flow` - подтверждает движение объемом.

## Фигуры

В контур нужно добавить и проверять:

1. Двойная вершина.
2. Двойное дно.
3. Голова и плечи.
4. Перевернутая голова и плечи.
5. Треугольник.
6. Вымпел.
7. Восходящий клин.
8. Нисходящий клин.
9. Диапазон.

## Логика входа

1. Найти фигуру на заданном окне.
2. Проверить, где цена относительно локального уровня.
3. Проверить пробой или ретест.
4. Проверить подтверждение объемом.
5. Сформировать вход только если связка согласована.
6. Калибровать `long_only` и `short_only` отдельно.

## Что калибровать

1. `pattern_window` - окно поиска фигуры.
2. `min_touches` - минимальное число касаний границ.
3. `breakout_threshold` - сила пробоя.
4. `retest_window` - окно ожидания ретеста.
5. `level_distance_threshold` - допустимая близость к уровню.
6. `volume_confirm_threshold` - порог объемного подтверждения.
7. `vol_z_window` - окно относительного объема.
8. `sl_buffer` - запас стопа.
9. `tp_ladder` - лесенка тейк-профитов.

## Выход

`SL`:

1. За границу фигуры.
2. За локальный уровень.
3. За FIBO.
4. Через ATR-buffer.

`TP`:

1. `TP1` - ближайший локальный уровень.
2. `TP2` - measured move фигуры.
3. `TP3` - FIBO extension, следующий уровень или RR target.

## Статус

Это пока ТЗ/гипотеза, не реализация.

До реализации этого контура блок `pattern` считается неполным по части классических фигурных паттернов.
