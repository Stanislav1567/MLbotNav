# Пользовательский интерфейс STAS9

Статус: `VSCODE_CODEX_INTERFACE_READY`.

## Назначение

`STAS9_INTERFACE` организует общение пользователя с `STAS9_SENTINEL` через VS Code и панель Codex. Отдельное приложение и отдельное консольное окно для основного режима не требуются.

Основная цепочка:

```text
Ярлык 🤖 STAS9 Assistant
-> MLbotNav_STAS9.code-workspace
-> VS Code
-> панель Codex
-> STAS9_SENTINEL
-> специализированный агент ON_DEMAND
```

Старый `START/start_STAS9.bat` сохранён как технический терминальный режим.

## Быстрый запуск

1. Открыть ярлык `🤖 STAS9 Assistant` на рабочем столе.
2. Дождаться открытия workspace и панели Codex.
3. Создать новый чат или продолжить текущий.
4. Начать команду со слов `STAS9` или `Sentinel`, например:

```text
STAS9, проверь реестр моделей и найди сломанные указатели.
```

5. Для голосового ввода установить курсор в поле сообщения, нажать `Win+H`, произнести команду по-русски, проверить полученный текст и отправить его.

## Состав

```text
CHAT/conversations/   — человекочитаемые сводки диалогов;
VOICE/input/          — только явно сохранённые входные voice/STT-артефакты;
VOICE/output/         — только явно сохранённые выходные артефакты;
COMMANDS/             — правила распознавания и маршрутизации команд;
RESPONSES/            — обязательный формат ответа.
```

Структурированные журналы диалогов находятся отдельно:

```text
STAS9_CONTROL_PLANE/LOGS/conversations/
```

## Ограничение голосового режима

Расширение Codex для VS Code принимает текст. В этой конфигурации преобразование речи выполняет системная диктовка Windows, а Sentinel получает уже распознанный текст. STAS9 не записывает микрофон и не сохраняет исходное аудио автоматически.
