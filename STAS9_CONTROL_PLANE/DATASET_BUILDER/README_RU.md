# Контур подготовки набора данных SOL

`DATASET_BUILDER` описывает безопасный и воспроизводимый путь:

```text
Свечи SOL → События → Признаки → Метки → ML dataset
```

## Этапы

1. Проверить источник свечей, временные метки, пропуски и дубли.
2. Найти интересную рыночную ситуацию и сохранить отдельное событие с происхождением данных.
3. Проверить событие и перевести его из кандидата в наблюдение.
4. Рассчитать признаки только из информации, доступной на момент события.
5. Назначить метку по версионированному правилу и заданному будущему горизонту.
6. Собрать строки, проверить схему, дубли, пропуски, баланс классов и временные утечки.
7. Сохранить паспорт версии набора данных и контрольную сумму артефакта.

## Контракты

- Событие обязано иметь уникальный идентификатор, статус, символ, таймфрейм и происхождение данных.
- Признаки обязаны иметь имя, тип, версию вычисления и момент доступности.
- Метка обязана ссылаться на версию правила, горизонт и измеримый результат.
- Строка набора обязана однозначно ссылаться на событие, контракт признаков и контракт метки.
- Шаблоны и непроверенные события не допускаются в итоговый набор.

## Ограничение первого этапа

На первом этапе разрешены только исследование свечей, сохранение событий, проектирование признаков, формализация меток и проверка будущего набора данных. Обучение моделей, подбор параметров и автоматическое изменение моделей не запускаются.

Машиночитаемый план и условия допуска определены в `SOL_DATASET_PIPELINE.yaml`.
