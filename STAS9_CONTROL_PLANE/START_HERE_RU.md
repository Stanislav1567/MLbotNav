# Как начать работу с STAS9 в новом чате Codex

## Короткий ответ

STAS9 не нужно запускать как отдельную программу. Откройте workspace
`MLbotNav_STAS9.code-workspace`, откройте панель Codex и отправьте стартовую
команду из этого документа.

Codex будет исполняющей средой, `STAS9_SENTINEL` — управляющей ролью, а
`STAS9_CONTROL_PLANE` — постоянной памятью, набором правил и журналов.

## 1. Откройте правильный workspace

Основной вариант:

```text
Рабочий стол -> 🤖 STAS9 Workspace
```

Или откройте файл:

```text
C:\Users\007\Desktop\MLbotNav\MLbotNav_STAS9.code-workspace
```

Если панель Codex не открылась автоматически:

```text
Ctrl+Shift+P -> Open Codex Sidebar
```

Terminal launcher `START/start_STAS9.bat` является только техническим резервом
и для обычной работы через чат не нужен.

## 2. Первая команда в новом чате

Пока команда не отправлена, STAS9 ничего не выполняет: отдельный агентский
процесс не работает, автоматическое сканирование каталогов и фоновая запись
журналов отключены.

Скопируйте и отправьте короткую команду готовности:

```text
STAS9, проверка готовности.
```

Ожидаемый результат: Sentinel подтверждает `SAFE`, `READ_ONLY` и готовность,
но ничего не изменяет и не запускает.

Для нового содержательного чата используйте расширенную команду:

```text
STAS9, начни новый рабочий чат.

Работай как STAS9_SENTINEL внутри Codex.
Режим по умолчанию: SAFE, доступ READ_ONLY.

Сначала прочитай:
1. STAS9_CONTROL_PLANE/CODEX_INSTRUCTIONS_RU.md
2. STAS9_CONTROL_PLANE/MEMORY/STAS9_STATE.md
3. STAS9_CONTROL_PLANE/MEMORY/PROJECT_MEMORY.yaml
4. STAS9_CONTROL_PLANE/MEMORY/AGENT_RESPONSIBILITIES.yaml
5. STAS9_CONTROL_PLANE/TASK_ROUTER_RU.md

Перед первой содержательной проектной задачей прочитай обязательные
docs/codex/handoff.md, current_state.md, todo.md, known_errors.md и commands.md.
Не загружай большие журналы, datasets, модели, caches и run-каталоги без
необходимости.

Подтверди активный режим, текущий статус и готовность. Ничего не изменяй.
```

Это стартовая проверка. Она не должна запускать полный аудит, обучение,
Optuna, эксперименты, специализированных агентов или изменение файлов.

## 3. Затем отправьте конкретную задачу

Безопасная проверка:

```text
STAS9, в режиме SAFE проверь <что именно>.
Ничего не изменяй.
Дай краткий вывод и укажи прочитанные источники.
```

Пример SOL-задачи:

```text
STAS9, проведи аудит данных SOL.

Режим: SAFE.
Ничего не изменяй.
Создай только отчёт.
```

Изменение файлов:

```text
STAS9, активируй DEVELOPMENT для задачи <точная задача>.
Разрешаю изменить только:
- <точный путь 1>
- <точный путь 2>

После изменений выполни:
- <точная проверка 1>
- <точная проверка 2>

STAS5–STAS8 не изменять.
После завершения вернись в SAFE.
```

Эксперимент:

```text
STAS9, активируй EXPERIMENT только для <точная гипотеза>.
Разрешённая область записи: <точный каталог STAS9_CONTROL_PLANE>.
Запрещены live trading, production pointers, перезапись моделей и удаление
артефактов. Обучение разрешено только отдельной прямой командой.
После завершения вернись в SAFE.
```

## 4. Как сохраняется непрерывность между чатами

После важной работы STAS9 должен кратко обновлять:

```text
STAS9_CONTROL_PLANE/MEMORY/STAS9_STATE.md
```

В нём сохраняются текущий статус, правила, известные ограничения и последние
действия. Структурированные сведения остаются в:

```text
STAS9_CONTROL_PLANE/MEMORY/PROJECT_MEMORY.yaml
```

Содержательные команды регистрируются отдельными YAML-файлами в:

```text
STAS9_CONTROL_PLANE/LOGS/conversations/
```

Большие отчёты и результаты не вставляются в память целиком: в ней остаются
только краткий вывод и путь к артефакту.

## 5. Важные ограничения

1. Новый чат не должен полагаться на память предыдущего чата — он заново
   читает файлы постоянной памяти.
2. Фактические файлы проекта имеют приоритет над текстом памяти.
3. `SAFE` не разрешает менять исходные проекты.
4. `DEVELOPMENT` и `EXPERIMENT` требуют явной ограниченной команды
   пользователя.
5. STAS5–STAS8, модели, legacy, `BROKEN_POINTER`, обучение, Optuna,
   production promotion и live trading не изменяются автоматически.
6. Все агенты одновременно не запускаются; специалисты используются только
   `ON_DEMAND`.
7. Закрытие задачи Codex завершает активную сессию роли; постоянный процесс
   STAS9 после этого не остаётся.

## 6. Голосовой ввод

Поставьте курсор в поле сообщения Codex, нажмите `Win+H`, продиктуйте команду,
проверьте распознанный текст и отправьте его. STAS9 получает обычный текст и
не сохраняет исходное аудио.

## 7. Ожидаемый ответ STAS9

Каждый итоговый ответ содержит:

```text
СТАТУС:
АГЕНТ:
ЗАДАЧА:
РЕЗУЛЬТАТ:
ИЗМЕНЕНИЯ:
```
