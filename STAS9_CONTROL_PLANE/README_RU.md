# STAS9 Control Plane

Статус: `STAS9_VSCODE_CODEX_ON_DEMAND_SAFE_READY`.

Дата аудита: `2026-07-23`.

## Назначение

STAS9 — управляющий проект над существующими контурами STAS1–STAS8. Он не копирует их данные, не переобучает модели и не меняет решения старых версий. Его задача — хранить карту проекта, реестры, правила архива и предлагаемую архитектуру следующего управляющего слоя.

Основной пользовательский интерфейс — workspace `MLbotNav_STAS9.code-workspace` в VS Code с установленной панелью Codex. Главная точка входа — `STAS9_SENTINEL` в режиме `INTERACTIVE_ASSISTANT_MODE`.

## Состав

1. `PROJECT_MAP.md` — аудит и карта STAS1–STAS8.
2. `MODEL_REGISTRY.yaml` — реестр логических моделей и модельных артефактов.
3. `FEATURE_REGISTRY.yaml` — реестр контрактов признаков.
4. `TASK_REGISTRY.md` — инвентаризация существующих технических заданий.
5. `ARCHIVE_POLICY.md` — правила чтения, повторного использования и запрета перезаписи старых данных.
6. `STAS9_ARCHITECTURE_PROPOSAL_RU.md` — предлагаемая архитектура STAS9.
7. `CODEX_INSTRUCTIONS_RU.md` — постоянные правила работы Codex как Sentinel.
8. `STAS9_INTERFACE/` — текстовый, голосовой, командный и ответный интерфейс.
9. `CONFIG/INTERACTION_MODES.yaml` — режимы `SAFE`, `DEVELOPMENT`, `EXPERIMENT`.
10. `LOGS/conversations/` — структурированный журнал общения.
11. `AGENT_ROLES/` — явное распределение обязанностей управляющих ролей.
12. `TASK_ROUTER_RU.md` — человекочитаемый маршрутизатор задач.
13. `MARKET_KNOWLEDGE/` — база знаний и шаблоны событий рынка SOL.
14. `DATASET_BUILDER/` — безопасный план преобразования событий в ML dataset.
15. `MEMORY/AGENT_RESPONSIBILITIES.yaml` — память делегирования и результатов.
16. `CODEX_WORKFLOW_RU.md` — основной сценарий VS Code + Codex без фонового
    процесса.
17. `START/README_RU.md` — различие между основным workspace и техническим
    terminal launcher.

## Запуск

Основной режим:

```text
Рабочий стол\🤖 STAS9 Workspace
-> MLbotNav_STAS9.code-workspace
-> VS Code + Codex
```

Роль Sentinel активируется только пользовательской командой и не работает
как постоянная служба или отдельный фоновый процесс.

Технический резерв:

```text
START/start_STAS9.bat
```

## Жесткая граница

При создании STAS9:

1. файлы и каталоги STAS5–STAS8 не изменяются;
2. обучение, forward, Optuna и торговый API не запускаются;
3. старые модели, данные, отчеты и run-артефакты не удаляются;
4. STAS9 начинает с read-only управления и проверки ссылок;
5. любое будущее изменение старого контура требует отдельного технического задания и явного разрешения.
