# Рыночные события SOL

Каждое событие хранится в отдельном YAML-файле с идентификатором вида `EVENT_SOL_000001`.

Допустимый жизненный цикл:

1. `TEMPLATE_NOT_OBSERVED` — только пример структуры, не рыночное наблюдение.
2. `CANDIDATE` — найденная ситуация, ожидающая проверки источника и результата.
3. `OBSERVED` — проверенное наблюдение с происхождением данных.
4. `LABELED` — проверенное наблюдение, которому назначена метка по утверждённым правилам.

В набор данных могут попадать только проверенные события. Файл `EVENT_SOL_000001.yaml` является шаблоном и не может использоваться как наблюдение.

Новый контракт событий SOL описан в
`../EVENT_SCHEMA_RU.md`. Будущие карточки `SOLUSDT 1m` должны храниться в
`SOL/` и проходить новый контракт до получения статуса наблюдения.

`EVENT_SOL_000001.yaml` остаётся legacy-шаблоном
`TEMPLATE_NOT_OBSERVED`: он не мигрирован в новый контракт, не является
событием и не должен использоваться как источник dataset.
