# База знаний рынка SOL

Назначение каталога — хранить проверяемые артефакты исследования свечей SOL и последовательно преобразовывать наблюдения в данные для ML.

## Структура

- `SOL/` — описание источников и области исследования данных SOL.
- `EVENT_SCHEMA_RU.md` — причинный контракт события, outcome, label и lineage.
- `EVENTS/` — отдельные карточки рыночных событий; `EVENTS/SOL/` зарезервирован
  для проверенных событий `SOLUSDT 1m`.
- `PATTERNS/` — повторяемые конфигурации свечей и контекста.
- `LABELS/` — определения целевых классов и правила разметки.
- `DATASETS/` — паспорта подготовленных наборов данных, но не сами исходные свечи.

## Правила

1. Каждое реальное событие должно ссылаться на конкретный источник данных и временной интервал.
2. Шаблоны нельзя считать наблюдениями или включать в набор данных.
3. Результат будущего движения фиксируется только после проверки горизонта события.
4. На первом этапе выполняются поиск, разметка и формализация событий; запуск обучения запрещён.
5. Исходные свечи и существующие артефакты не изменяются и не удаляются.
6. Единственный источник текущего SOL-контура —
   `data/core/bybit_ohlcv`; BTC и смешивание физических копий запрещены.
