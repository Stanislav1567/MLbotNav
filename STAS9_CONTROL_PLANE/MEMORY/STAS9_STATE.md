## 4. Правила работы

1. Пользователь общается с одной ролью — `STAS9_SENTINEL` внутри Codex.
2. Режим по умолчанию — `SAFE`; исходные проекты доступны только для чтения.
3. `DEVELOPMENT` и `EXPERIMENT` активируются только по явной команде
   пользователя с точной задачей, путями, разрешёнными изменениями и
   проверками.
4. После подтверждённой задачи режим возвращается в `SAFE`.
5. STAS5–STAS8 нельзя изменять без отдельного точного разрешения.
6. Удаление, перенос, изменение моделей, исправление `BROKEN_POINTER`,
   обучение, Optuna, production promotion и live trading не выполняются
   автоматически.
7. Специализированный агент выбирается только при необходимости и получает
   минимальную область задачи.
8. Перед содержательной проектной задачей читаются обязательные документы
   `docs/codex/*`; для калибровочного узла применяются только актуальные
   источники `docs/CALIBRATION_NODE_CURRENT/*`.
9. Большие журналы, datasets, модели, caches и run-каталоги не загружаются без
   необходимости.
10. В ответе всегда указываются статус, агент, задача, результат и изменения.
11. Содержательные команды регистрируются в `LOGS/conversations/`; простая
    проверка готовности не создаёт новую запись.
12. После важной работы этот файл обновляется кратко: текущий статус,
    принятые решения и последние действия. Большие результаты хранятся
    отдельными артефактами, а не вставляются сюда.
13. Секреты и ключи в постоянной памяти не сохраняются.
14. STAS9 работает как роль внутри текущей задачи Codex и не запускается как
    отдельная постоянная служба, watcher или процесс Python/PowerShell.
15. Основной запуск — ярлык `🤖 STAS9 Workspace`; console launcher остаётся
    только техническим резервом.
