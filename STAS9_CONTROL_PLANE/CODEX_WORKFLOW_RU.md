# Рабочий процесс STAS9 в VS Code и Codex

Статус: `ACTIVE_ON_DEMAND_SAFE`.

## 1. Основная схема

```text
Пользователь
  -> MLbotNav_STAS9.code-workspace
  -> VS Code и панель Codex
  -> обращение к STAS9
  -> STAS9_SENTINEL как управляющая роль Codex
  -> профильный агент только ON_DEMAND
  -> краткий результат и запись в память проекта
```

`STAS9_CONTROL_PLANE` хранит постоянные правила, состояние, маршрутизацию,
отчёты и журналы. Он не запускает самостоятельную службу.

## 2. Запуск

1. Открыть на рабочем столе ярлык `🤖 STAS9 Workspace`.
2. Дождаться открытия `MLbotNav_STAS9.code-workspace` в VS Code.
3. Открыть панель Codex, если она не открылась автоматически.
4. Отправить команду, начинающуюся с `STAS9` или `STAS9_SENTINEL`.

Без пользовательской команды STAS9 не выполняет аудит, поиск, мониторинг,
чтение datasets или запуск специализированных агентов.

## 3. Выполнение команды

1. Sentinel принимает и классифицирует задачу.
2. Проверяет режим и разрешённую область.
3. Выбирает профильного агента по `TASK_ROUTER_RU.md`.
4. Запускает только необходимую работу в текущей задаче Codex.
5. Проверяет результат и возвращает ответ по шаблону STAS9.
6. Для содержательной команды создаёт одну событийную запись в
   `LOGS/conversations/`.

Специалисты не работают постоянно и не запускаются все одновременно.

## 4. Режимы и права

- по умолчанию: `SAFE`;
- доступ по умолчанию: `READ_ONLY`;
- выполнение: `ON_DEMAND`;
- триггер: `USER_COMMAND_ONLY`;
- действие в ожидании: `NONE`;
- фоновый процесс, watcher и автоматическое сканирование: отключены;
- `DEVELOPMENT` и `EXPERIMENT`: только по отдельной явной команде
  пользователя с точной областью изменений.

После подтверждённой изменяющей задачи STAS9 возвращается в `SAFE`.

## 5. Память и журналы

Постоянное состояние хранится в:

```text
MEMORY/STAS9_STATE.md
MEMORY/PROJECT_MEMORY.yaml
MEMORY/AGENT_RESPONSIBILITIES.yaml
```

Логи создаются по событию: одна завершённая содержательная команда — одна
запись. Простой запуск VS Code, открытие панели Codex, приветствие и ожидание
не создают периодических записей.

## 6. Технический резерв

`START/start_STAS9.bat` оставлен для диагностики и ручной работы с Codex CLI.
Это не основной интерфейс и не автозапускаемый фоновый компонент.

## 7. Ресурсная дисциплина

Workspace использует один корень `MLbotNav`, поэтому видны управляющий слой,
`STAS5_ML_CORE` и вложенные материалы STAS8. Тяжёлые каталоги данных, моделей,
артефактов, кешей и запусков исключены из фонового поиска и наблюдения VS
Code. Защитные исключения Windows Defender автоматически не изменяются.

STAS9 не запускает постоянные процессы Python, PowerShell, watcher, обучение,
Optuna или реальную торговлю.
