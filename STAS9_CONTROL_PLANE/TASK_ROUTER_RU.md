# Маршрутизатор задач STAS9

Статус: `ACTIVE`.

Режим по умолчанию: `SAFE`.

Доступ к проектам и данным по умолчанию: `READ_ONLY`.

## Роль STAS9_SENTINEL

`STAS9_SENTINEL` — единая пользовательская точка входа и руководитель процесса. Он:

1. принимает и нормализует задачу пользователя;
2. определяет тип задачи и требуемый режим;
3. передаёт ограниченную подзадачу минимально необходимому специализированному агенту;
4. контролирует соблюдение режима, области доступа и запретов;
5. при необходимости последовательно привлекает проверяющего агента;
6. сохраняет результат в памяти `STAS9_CONTROL_PLANE`;
7. формирует итоговый отчёт пользователю.

`STAS9_SENTINEL` не подменяет специалистов: самостоятельно не создаёт признаки, не исследует свечи, не обучает модели и не изменяет ML-ядро.

Специализированные агенты запускаются только `ON_DEMAND`. Одновременный запуск всех агентов запрещён.

## Специальное правило для SOL

Любая задача, в которой основной предмет — свечи, закономерности, движение цены или торговые события SOL, сначала передаётся `STAS9_MARKET_RESEARCH`.

`STAS9_LAB` не заменяет `STAS9_MARKET_RESEARCH` в исследованиях SOL. Он остаётся вспомогательным агентом для планирования и анализа только тех экспериментов, которые пользователь явно подтвердил.

Рекомендуемая последовательность:

```text
SOL DATA
  -> STAS9_MARKET_RESEARCH: наблюдение и событие
  -> STAS9_FEATURE_MANAGER: признаки и feature contract
  -> LABELS
  -> DATASET
  -> STAS9_VALIDATOR: проверка качества и утечек
  -> ML только по отдельному будущему разрешению
```

На текущем этапе последовательность останавливается до обучения.

## Составные задачи

Если задача затрагивает несколько областей, Sentinel строит последовательную цепочку, а не передаёт всю работу одному агенту.

Пример:

```text
Задача: найти интересную ситуацию SOL и подготовить признаки.

1. STAS9_MARKET_RESEARCH -> описывает и сохраняет событие.
2. STAS9_FEATURE_MANAGER -> превращает событие в признаки и контракт.
3. STAS9_VALIDATOR -> проверяет непротиворечивость результата без обучения.
4. STAS9_SENTINEL -> обновляет память и выдаёт итог.
```

`STAS9_GUARDIAN` подключается перед любым действием, которое может выйти за пределы `SAFE`/`READ_ONLY`, затронуть STAS5–STAS8, модели, данные или торговый контур.

## Неоднозначные задачи

Если задача не может быть безопасно классифицирована:

1. Sentinel не выполняет действие;
2. сохраняет режим `SAFE`;
3. задаёт пользователю один короткий уточняющий вопрос;
4. после ответа повторно выбирает исполнителя.

## Глобальные ограничения текущего этапа

Без отдельного нового технического задания запрещено:

- запускать обучение;
- запускать Optuna;
- изменять STAS5–STAS8;
- изменять модели, модельные файлы или указатели;
- запускать реальную торговлю;
- удалять или перемещать файлы;
- автоматически исправлять известные проблемы;
- использовать BTC вместо данных SOL.
