# ТЗ: Восстановление ошибок кодировки и полный аудит кода/файлов

Дата: 2026-05-24  
Статус: ACTIVE  
Область: весь проект `MLbotNav` (код, скрипты, docs, AKFP, отчеты/артефакты).

## 1. Цель
1. Восстановить поврежденные текстовые блоки (битая кириллица, «последовательности вопросительных знаков») без потери смысла и нумерации.
2. Провести полный аудит кода/скриптов/пакетов/файлов и устранить подтвержденные техриски.
3. Зафиксировать защиту от повторного появления проблем кодировки.

## 2. Подтвержденные проблемы (база аудита)
1. Повреждение текста в ряде русскоязычных документов.
2. Техриски в инфраструктурных скриптах:
   - жесткий `Set-Location` на абсолютный путь,
   - запуск через `.venv\Scripts\python.exe` без precheck наличия `.venv`.
3. Недостающие зависимости для отдельных режимов runtime (ранее закрыто на шаге R3).

1. `reports/qa_gate/tz_plan_text_encoding_audit_20260524T182505Z.json`

## 4. План работ (строго по шагам)

### R1. Forensic-аудит кодировки
1. Собрать список файлов/строк с повреждениями.
2. Разделить на:
   - восстанавливается автоматически по контексту,
   - требует ручного восстановления.
3. Артефакт: `reports/qa_gate/recovery_r1_encoding_forensic_*.json`.
4. PASS-критерий: полный список целей восстановления с диапазонами.

### R2. Восстановление документов (без изменения бизнес-логики)
1. Восстановить нормальный русский текст в целевых `*.md`.
2. Сохранить исходную структуру и нумерацию разделов.
3. Артефакт: `reports/qa_gate/recovery_r2_docs_restore_*.json`.
4. PASS-критерий: в целевых диапазонах нет битой кириллицы/последовательностей вопросительных знаков.

### R3. Техфикс зависимостей и окружения
1. Добавить недостающие runtime/test зависимости.
2. Проверить import-smoke и `pytest --version`.
3. Артефакт: `reports/qa_gate/recovery_r3_deps_*.json`.
4. PASS-критерий: модульные импорты проходят, зависимость воспроизводима.

### R4. Harden `*.ps1` запускателей
1. Убрать абсолютный `Set-Location "C:\Users\007\Desktop\MLbotNav"`.
2. Перевести на корень от `$PSScriptRoot`.
3. Добавить precheck `.venv\Scripts\python.exe` там, где используется локальный Python.
4. Артефакт: `reports/qa_gate/recovery_r4_ps1_hardening_*.json`.
5. PASS-критерий: скрипты портативны и дают понятную ошибку при отсутствии `.venv`.

### R5. Anti-regression text guard
1. Добавить проверку текстовой целостности (битая кириллица/последовательности вопросительных знаков).
2. Включить проверку в ежедневный аудиторский контур.
3. Артефакт: `reports/qa_gate/recovery_r5_text_guard_*.json`.
4. PASS-критерий: guard валит поврежденный текст и PASS на чистом состоянии.

### R6. Финальный полный аудит
1. Прогнать цепочку compile/import/ps1-parse + table/qa gates.
2. Подтвердить, что восстановленные файлы читаемы.
3. Артефакт: `reports/qa_gate/recovery_r6_final_audit_*.json`.
4. PASS-критерий: обязательные проверки PASS и нет новых повреждений.

## 5. Правила выполнения
1. После каждого шага статус только `PASS` или `FAIL`.
2. Если шаг `FAIL` — фикс в этом же шаге, без перескока.
3. Изменения делаем в текущем проекте без раздувания структуры.
4. В отчете шага обязательно:
   - что внедрено,
   - какие команды запускались,
   - ссылки на артефакты.

## 6. Definition of Done
1. В целевых документах нет битых блоков.
2. По коду/скриптам нет критичных неисправленных рисков.
3. Итоговый аудит PASS.
4. Есть обновленный читаемый журнал восстановления.
